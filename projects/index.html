Moka A powerful tool to enhance in-room collaboration

  • Android application and central web interface
  • developed between March and June 2013
  • uses Java, HTML5 and Javascript, REST, Jade
  • source available on Github

The Moka project intends to change the way software engineers conceive their products and regroup their ideas.


After having taken place in front of a single large screen, the attendants of a conception meeting can interact with the web interface displayed on that screen, using their Android smartphone. They can add or edit content such as UML diagrams, images, web links and so on. At the end of the meeting, the content displayed on the screen can be saved for later reuse.

The goal of Moka is to simplify in-room collaboration. No need for any type of expensive hardware such as interactive whiteboards. One video projector, one computer and each attendant's phone is everything you need to start creating things efficiently.

If you want to learn more about Moka, just visit the website right there.

UVweb 2.0 UTC's exchange plateform reinvented

  • Web app and Android application
  • development started in December 2012
  • the web-app uses PHP with Symfony2, and advanced CSS with Twitter Bootstrap
  • the Android app uses Java
  • source available on Github

UVweb is a school-sized social network allowing students from my university (the UTC) to give their opinion on the lectures they've attended.

Until december 2012, Thomas Keunebroek and I were just admins of the website, but we then decided to rebuild it from scratch. We'll like the new UVweb to be smartest, easier to use and to have a responsive design. And of course, as Android lovers, we can't help but develop an Android app for phones and tablets with it.

You can follow the project's advancement by checking out the Github pages of the web-app and the Android app.

We hope UVweb 2 to be operational by the end of June 2013. Until then, check out the current version of the platform !

RI02 Expert a Decision Helping Expert System for Erasmus students

  • Two inference engines based script
  • developped between November 2011 and January 2012
  • uses Common LISP
  • source available on Github

This Expert System is a prototype of a computerizing tool. It was created to help the Outgoing Department of the Université de Technologie de Compiègne in the whole student selecting process for a semester of studies abroad.

The system, written in Common Lisp, is based on two inference engines.

The first one uses forward-chaining. It starts the deducing process by the goal : a matching student profile for a semester in another university. It then asks questions based on what can be helpful or not for the student to go to the university.

The second one, backward-chaing based, starts by the student's profile and asks dynamically the good questions to try and deduce his affinity with the university.

Although the code is a bit dense, you can take a look at it here.
